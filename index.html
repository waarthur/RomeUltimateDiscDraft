🗄️ Supabase Database Setup
✅ Connected to Supabase! Registrations are stored in the cloud and visible from any device.
⚠ Database Not Yet Connected

Registrations are currently only stored locally on this device. Follow these 4 steps (≈5 minutes) to enable cloud storage so anyone can register from any device.

1
Create a free Supabase account
Go to supabase.com → Start your project. Sign up with GitHub or email — it's free forever on the hobby tier.
2
Create a new project
Click New Project. Name it discdraft. Set a database password and choose the region nearest to you. Click Create new project and wait about 60 seconds.
3
Run this SQL to create the players table
In your project click SQL Editor in the left sidebar. Paste the SQL below and click Run: 
create table players (
  id bigint generated always as identity primary key,
  first text not null,
  last text not null,
  age integer not null,
  email text,
  talent integer not null,
  event_id text not null,
  registered_at timestamptz default now()
);

alter table players enable row level security;

create policy "Public read" on players
  for select using (true);

create policy "Public insert" on players
  for insert with check (true);

create policy "Public delete" on players
  for delete using (true);
4
Copy your API credentials
Go to Project Settings (gear icon, bottom-left) → API. Copy your Project URL and the anon / public key. Paste them in the fields below, then click Test Connection.
⚙️ Event Settings
📥 After filling in your settings, click Save & Download index.html. Upload that file to GitHub Pages — it will have your database credentials and event details baked in, so every visitor connects automatically with no login required.
🗑️ Manage Roster

Remove individual players or wipe the entire roster from the database.

🔗 Deploy & Share
💡 To give players a shareable link, host this file for free on GitHub Pages. Follow the steps below.
1
Create a free GitHub account
Go to github.com and sign up. Confirm your email address.
2
Create a new public repository
Click + → New repository. Name it discdraft. Set to Public. Check Add a README. Click Create repository.
3
Upload this file renamed as index.html
Click Add file → Upload files. Upload discdraft.html renamed to index.html. Click Commit changes.
4
Enable GitHub Pages
Go to Settings → Pages. Source: Deploy from branch. Branch: main, folder: / (root). Click Save. Your URL appears after ~60 seconds.
5
Paste your URL below to generate a group chat message
Your URL will look like: https://yourname.github.io/discdraft/
